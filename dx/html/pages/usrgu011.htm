New Image Formats Available

With the ImageMagickTM libraries configured and compiled with DX, the ReadImage and WriteImage modules can read and write a large number of image file formats. ImageMagick is a robust collection of tools and libraries to read, write, and manipulate an image in many image formats (over 68 major formats) including popular formats like TIFF, JPEG, PNG, PDF, PhotoCD, and GIF. The formats that are available to DX will depend on what functionality is compiled into ImageMagick.

In order to access the new image formats, select the format as "ImageMagick supported format" and add an extension to the filename that ImageMagick recognizes within the ReadImage and WriteImage modules and Save Image dialog box.

Although DX still supports TIFF and MIFF files with its own code, it is possible to have ImageMagick write these formats as well. Some of the default compression settings on these formats may be different within ImageMagick. Instead of selecting TIFF or MIFF in the format box, use appropriate extension and select "Image Magick supported format" as the format to get ImageMagick to write a miff file. The DX .rgb and ImageMagick .rgb are completely different formats. Due not confuse them.

ReadImage with the ImageMagick libraries supports:

  • creating opacities/opacity map components for images which have opacities or a transparency map, such as the GIF format and PNG format,
  • both delayed and direct color images, with delayed->direct option and float/byte option,
  • smart chomping of supported ImageMagick extensions
  • the DX convention for reading multi-frame multi-file sequences; such as name="t.gif", where the image files are t.0.gif, t.1.gif, t.2.gif, ...,
  • for multi-frame single-file images such as animated GIFs, stacked TIFF/PNGs, but only loading the first image is supported (ImageMagick is missing basic sub-image support; see the source comments for details).

Due to patent licensing issues, DX removed support for the gif format directly but with the ImageMagick extension, DX supports this format without adding the patent issue problems.
