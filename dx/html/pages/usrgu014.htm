Networking Enhancements

The default command for remoted communication between executives or between the UI and the executive, rsh, has been left in place, but a user-specified remote shell command can be specified via the DXRSH environment variable. For example: setenv DXRSH /usr/bin/ssh, would set up DX to use secure shell for communication.

FQDNs (full host paths) are used under the hood when available when creating the remote "invoke dxexec" script and connecting back to the master exec. This may cause problems for non-networked machines who may want to set the DXHOST environment variable.

Users can now enter host names of the form "user@machine" in the Execute Group Assignment... dialog adding functionality to the ssh networking feature, and

setting the DXRSH_NOENV in the local environment prevents the remote "invoke dxexec" script from being written with a full carbon copy of the local environment (only DISPLAY is set).
